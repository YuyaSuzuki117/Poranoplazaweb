## デザイン方針

### 美的方向性: インダストリアル・プレミアム
- **トーン**: 洗練されたインダストリアル - 内装工事のクラフトマンシップを表現
- **差別化ポイント**: 施工写真を建築雑誌のようなエディトリアルレイアウトで見せる
- **印象**: プロフェッショナル、信頼、技術力、モダン

### タイポグラフィ
- 見出し: 個性的なディスプレイフォント（Google Fontsから選定）
- 本文: Noto Sans JP（可読性重視）
- text-balance（見出し）、text-pretty（本文）を適用
- tabular-nums（数値データ）
- letter-spacing は変更しない

### カラーシステム
- CSS変数で一元管理
- 1ビューにつきアクセントカラーは1色まで
- 紫グラデーション禁止、グラデーション使用は明示的指示時のみ
- グロー効果を主要アフォーダンスにしない
- 既存テーマ / Tailwind デフォルトカラートークン優先

### レイアウト
- Bento Grid（モジュラーレイアウト）を活用
- 非対称・オーバーラップ・グリッドブレイク要素
- 施工写真を大きく見せるエディトリアルレイアウト
- z-index は固定スケール（任意の z-* 禁止）
- h-screen 禁止 → h-dvh を使用
- 正方形要素は size-* を使用

### アニメーション・モーション
- 明示的に要求された場合のみアニメーション追加
- transform と opacity のみアニメーション可（コンポジタプロパティ）
- layout プロパティ（width, height, top, left, margin, padding）のアニメーション禁止
- インタラクションフィードバックは 200ms 以内
- エントランスは ease-out
- prefers-reduced-motion を尊重
- 画面外のループアニメーションは一時停止
- 大きな blur / backdrop-filter のアニメーション禁止
- will-change はアクティブアニメーション中のみ

### 背景・ビジュアル
- ソリッドカラーデフォルトではなく、深みと雰囲気を演出
- ノイズテクスチャ、幾何学パターン、コンクリート質感など文脈に合った表現
- カスタム写真を信頼構築の中心に据える

## コーディング規約

### CSS
- Tailwind CSS デフォルト値を使用（カスタム値は既存 or 明示的要求時のみ）
- cn ユーティリティ（clsx + tailwind-merge）でクラスロジック管理
- Tailwind デフォルトシャドウスケール使用

### HTML
- セマンティックHTML5タグ（header, nav, main, section, footer）
- ネイティブ要素優先（button, a, input）- role ベースのハックより優先
- 見出しレベルをスキップしない
- リストは ul/ol + li

### アクセシビリティ（WCAG準拠）
- 全インタラクティブ要素にアクセシブル名
- アイコンのみボタンは aria-label 必須
- 全 input/select/textarea にラベル必須
- Tab で全インタラクティブ要素到達可能
- tabindex > 0 禁止
- モーダルはフォーカストラップ、Escape で閉じる
- エラーは aria-describedby でフィールドに紐付け
- required フィールドはアナウンス
- コントラスト比 4.5:1 以上
- hover のみインタラクション禁止（キーボード代替必須）
- 画像 alt 必須（装飾画像は aria-hidden）
- prefers-reduced-motion 対応

### メタデータ（SEO）
- 全ページに title + meta description
- canonical URL 設定
- OGP（og:title, og:description, og:image）設定
- twitter:card 設定（summary_large_image）
- html lang="ja"
- favicon + apple-touch-icon
- JSON-LD 構造化データ（実在コンテンツのみ）
- 重複メタタグ禁止
- og:url と canonical を一致させる

### パフォーマンス
- 画像: WebP 優先、適切な srcset/sizes
- will-change はアクティブアニメーション外で使用禁止
- 大きな blur() / backdrop-filter アニメーション禁止
- useEffect はレンダーロジックで表現できるなら使わない
- 軽量・高速なページロード

### フォーム
- input/textarea で paste をブロックしない
- AlertDialog で破壊的アクション確認
- エラーはアクション発生箇所に表示
- ローディングは構造スケルトン推奨
- XSS 対策を徹底

## 作業ルール
- 変更前に必ず既存コードを読んでから編集する
- 1つの作業単位ごとにタスクリストで管理する
- 大きな変更はプランモードで設計してから実装する
- ファイル作成は最小限に、既存ファイルの編集を優先する
- セキュリティ（XSS等）に注意してフォーム実装する
- コンポーネントプリミティブは既存のものを優先使用
- アニメーションライブラリの移行はユーザー明示的要求時のみ
